普通字符

匹配时不作改变

  • 例如：字符串”bcd”，在匹配”abcde”时匹配成功，匹配到的位置是：开始于1，结束于4。

简单的转义字符

  1. 一些不便书写的字符

    • /t 制表符
    • // 代表/本身
    • /r和/n 回车和换行

  2. 有特殊用处的符号

    • /$ $
    • /. .
    • /^ ^

  3. 可以和多个字符匹配的表达式

    • /d 0~9 中的任意一个数字
    • /w 任意一个字母或数字或下划线，也就是 A~Z,a~z,0~9,_ 中任意一个
    • /s 包括空格、制表符、换页符等空白字符中的任意一个
    • . 可以匹配除了换行符（/n）以外的任意一个字符
    • 举例：表达式 “/d/d”，在匹配 “abc123” 时，匹配的结果是：成功；匹配到的内容是：”12”；匹配到的位置是：开始于3，结束于5。

  4. 自定义能够匹配多种字符的表达式

    • {ab5@} 匹配 “a” 或 “b” 或 “5” 或 “@”
    • {^abc}匹配 “a”,”b”,”c” 之外的任意一个字符
    • {f-k}匹配 “f”~”k” 之间的任意一个字母
    • {^A-F0-3}匹配 “A”~”F”,”0”~”3” 之外的任意一个字符
    • 举例1：表达式 “{bcd}{bcd}” 匹配 “abc123” 时，匹配的结果是：成功；匹配到的内容是：”bc”；匹配到的位置是：开始于1，结束于3。
    • 举例2：表达式 “{^abc}” 匹配 “abc123” 时，匹配的结果是：成功；匹配到的内容是：”1”；匹配到的位置是：开始于3，结束于4。

  5. 重复匹配

    • 比如：”{bcd}{bcd}” 可以写成 “{bcd}{2}”
    • {n} 表达式重复n次，比如”a{5}”表示”aaaaa”
    • {m,n} 表达式至少重复m次，至多重复n次
    • {m,} 表达式至少重复m次
    • ？ 表达式重复0或1次，相当于{0,1}
    • /+ 表达式重复1次以上
    • /* 表达式重复0次以上

  6. 特殊符号

    • ^ 匹配开始位置，本身不匹配符号
    • $ 匹配结束位置，本身不匹配符号
    • /b 匹配一个单词边界，也就是单词和空格之间的位置,本身不匹配符号

  7. 影响表达式内部关系的符号

关于Git

Share

1、Git基础——常用命令

  • git clone 地址 //将远程代码复制到本地
  • git pull //取回远程主机某个分支的更新，再与本地的指定分支合并。
  • git add (filename) //添加文件入本地库（一个个添加）
  • git rm filename //删除文件
  • git commit -m “提示信息” //添加本地文件入缓冲区（批量）
  • git push -u origin 分支名 //将缓冲区（本地分支）的内容提交到远程仓库（origin）（批量操作）
  • git status //展示工作区的状态
  • git remote -v //查看远程库的信息
  • git init // 把当前文件夹变成一个本地git库
  • touch filename // 新建文件
  • git remote add origin（自定义名字）地址 //将git项目地址取名为origin添加到远程库,r然后可用git push -u origin master命令

2、Git分支

创建、查看与切换
  • git branch -a //查看所有分支
  • git branch 分支名 //在master下创建分支
  • git checkout 分支名 //切换分支，切换到不同分支时，本地文件夹的内容也不同。
分支合并、冲突
  • 分支状态下合并到主干的步骤：
    1. git merge master
    2. git checkout master
    3. git merge 分支名
    4. git push -u origin master
  • 当两个分支都改动了同一个部分时，会发生冲突，后提交到主干的人需要手动确认内容，解决冲突
代码（版本）回溯
  1. git log -l 数字 // 查看前几步的操作
  2. 复制操作记录码
  3. git reset -hard 操作记录码 // 本地库强制退回到先前记录
  4. git push -f -u origin 分支名 // 强制提交，删除记录
    或者git revert 操作记录码 // 安全提交，保留记录
同时推送到多个远程库
  1. git add 库名 库地址
  2. git push -u 库名 分支名
